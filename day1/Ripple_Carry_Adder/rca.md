# 4-Bit Ripple Carry Adder (RCA)

## Overview

A **Ripple Carry Adder (RCA)** is a combinational logic circuit used to perform binary addition of two multi-bit numbers. It is constructed by connecting multiple **Full Adders** in series, where the carry output of one stage becomes the carry input of the next stage.

In this project, a **4-bit Ripple Carry Adder** is implemented using four cascaded Full Adders. The design accepts two 4-bit operands and an input carry, producing a 4-bit sum and a final carry output.

---

## Design Architecture

The 4-bit RCA consists of four Full Adder stages connected sequentially.

### Block Diagram

```text
          A[0]  B[0]      A[1]  B[1]      A[2]  B[2]      A[3]  B[3]
            |     |          |     |          |     |          |     |
            v     v          v     v          v     v          v     v
         +---------+      +---------+      +---------+      +---------+
Cin ---> |  FA0    | ---> |  FA1    | ---> |  FA2    | ---> |  FA3    | ---> Cout
         +---------+      +---------+      +---------+      +---------+
              |                |                |                |
            Sum[0]          Sum[1]          Sum[2]          Sum[3]
```

The carry generated by each Full Adder propagates to the next stage. This propagation of carry bits through all stages gives the architecture its name **Ripple Carry Adder**.

---

## Working Principle

### Least Significant Bit (LSB) Stage

The first Full Adder adds:

```text
A[0] + B[0] + Cin
```

and generates:

```text
Sum[0] and Carry[0]
```

### Intermediate Stages

Each subsequent Full Adder adds:

```text
A[i] + B[i] + Carry[i-1]
```

where the carry output from the previous stage acts as the carry input.

### Most Significant Bit (MSB) Stage

The final Full Adder produces:

```text
Sum[3]
```

and the final carry output:

```text
Cout
```

---

## Propagation Delay

One limitation of the Ripple Carry Adder is carry propagation delay.

Since each Full Adder must wait for the carry signal from the previous stage, the total delay increases linearly with the number of bits.

For an N-bit Ripple Carry Adder:

```text
Total Delay ≈ N × Full Adder Delay
```

As a result, RCAs are simple and area-efficient but slower than advanced adders such as:

* Carry Lookahead Adder (CLA)
* Carry Select Adder (CSA)
* Carry Skip Adder (CSKA)
